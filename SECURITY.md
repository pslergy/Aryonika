# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly by contacting the maintainers directly.  

- **Email:** pslergy@gmail.com 


Please include as much detail as possible, including steps to reproduce the issue. Do not create public issues for security vulnerabilities.

## Response Process

Once a vulnerability is reported:

1. We will acknowledge your report within 48 hours.
2. We will investigate the issue and determine a fix.
3. We will coordinate a disclosure timeline and notify you when the fix is released.

## Supported Versions

We only provide security fixes for the latest stable version of the project.  

## Reporting Security Issues in Dependencies

If you find a security problem in a dependency, please report it to the maintainers of that dependency.
